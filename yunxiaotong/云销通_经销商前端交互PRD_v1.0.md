# 云销通·经销商前端平台 前端交互 PRD v1.0

> **文档用途**：驱动代码 Agent 自动生成云销通经销商端前端页面  
> **版本**：v1.0  
> **日期**：2026-07-16  
> **适用范围**：云销通经销商端全部前端页面（不含 OMS 内部运营后台）  
> **边界声明**：本文档仅定义面向经销商的前端交互层；所有审核、拣货、调度、数据聚合等内部运营逻辑归属 OMS，云销通通过 API 透传请求，不做本地业务判断  
> **变更摘要**：首版发布，确立淘宝式电商交互范式，覆盖首页浏览、智能购物车、结算下单、订单服务枢纽、对账财务、消息中心、个人中心共 7 大模块

---

## 1. 产品定位与设计原则

| 维度 | 说明 |
|------|------|
| **产品定位** | 面向授权经销商的 B2B 配件在线订购平台，提供类 C 端电商体验的采购全流程 |
| **目标用户** | 品牌授权经销商（门店老板、采购员、维修技师） |
| **设计范式** | 淘宝式电商体验：商品货架浏览 → 加购 → 智能购物车 → 结算 → 订单服务枢纽 |
| **视觉基线** | 橙色主调 + 卡片式布局，沿用现有云销通视觉体系，降低老用户迁移成本 |
| **系统边界铁律** | 云销通 = 纯交互层；所有写操作透传 OMS API；禁止本地缓存价格/库存/状态 |
| **信息密度** | B2B 场景优先展示物料编码、适配车型、阶梯价、库存分布等专业字段 |
| **录入原则** | 精简录入，VIN 码自动匹配车型，历史地址/发票信息自动回填 |

## 2. 信息架构总览

### 2.1 一级导航结构（共 7 个模块）

| 序号 | 模块名称 | 核心职责 | 对应 OMS 接口域 |
|------|---------|---------|----------------|
| 1 | 首页 / 商品浏览 | 品类导航、商品搜索、货架浏览、VIN 码匹配 | Product API / Vehicle Match API |
| 2 | 智能购物车 | 商品暂存、补货推荐、运费凑单、VIN 校验、替代件替换 | Cart API / Recommendation API |
| 3 | 结算下单 | 收货地址、配送方式、发票信息、订单确认、支付 | Order API / Payment API |
| 4 | 订单中心 | 订单列表、订单详情（服务枢纽）、售后申请、物流追踪 | Order API / AfterSales API / Logistics API |
| 5 | 对账 & 财务 | 对账单查看、差异反馈、发票管理、回款记录 | Reconciliation API / Invoice API |
| 6 | 消息中心 | 订单状态变更通知、促销推送、系统公告、客服消息 | Notification API / Chat API |
| 7 | 个人中心 | 账户信息、收货地址簿、发票信息管理、操作偏好 | User API / Address API / Preference API |

### 2.2 全局布局规范

| 区域 | 内容 | 交互说明 |
|------|------|---------|
| **顶部全局栏** | Logo + 搜索框 + 购物车图标(数量角标) + 消息铃铛 + 用户头像下拉菜单 | 固定吸顶；搜索框支持物料编码/VIN码/品名联想；购物车角标实时更新 |
| **品类导航条** | 水平排列的一级品类标签 | 置于搜索框下方，点击切换商品列表；选中态橙色高亮 |
| **主体内容区** | 各模块页面内容 | 最大宽度 1440px，居中布局；左右留白自适应 |
| **底部版权栏** | 公司名称、备案号、客服电话、在线客服入口 | 静态信息，常驻底部 |

### 2.3 搜索能力规格

| 搜索类型 | 触发条件 | 联想结果 | 跳转行为 |
|---------|---------|---------|---------|
| 物料编码 | 输入纯数字或"WL"前缀 | 精确匹配商品卡片（图+名+价格+库存） | 点击直达商品详情 |
| VIN 码 | 输入 17 位车架号 | 自动解析车型 → 展示该车型适配配件列表 | 进入车型专属商品页 |
| 品名关键词 | 输入中文/英文品名 | 模糊匹配商品列表（按相关度排序） | 进入搜索结果页 |
| 混合搜索 | 无法识别类型时 | 同时展示商品+车型+文章等多维结果 | 分 Tab 展示 |

## 4. 智能购物车模块

### 4.1 定位与核心理念

| 维度 | 说明 |
|------|------|
| **定位** | 不是简单的商品暂存区，而是"智能下单操作台" |
| **核心价值** | 帮经销商更快、更准、更省地完成批量采购决策 |
| **区别于 C 端** | 集成 VIN 校验、补货推荐、运费凑单、替代件替换等 B2B 专属能力 |

### 4.2 购物车商品列表

| 列字段 | 内容 | 交互 |
|--------|------|------|
| 勾选框 | 全选 / 单选 | 支持批量操作（删除、移入收藏） |
| 商品缩略图 | 64×64 白底图 | 点击跳转详情 |
| 商品信息 | 名称 + 物料编码 + 适配车型标签 | 物料编码可复制 |
| 单价 | 当前等级价（含税） | 数量变动时实时重算小计 |
| 数量 | 步进器 + 手动输入 | 低于最小起订量时红色警告 |
| 小计 | 单价 × 数量 | 含折扣标识（如有促销） |
| 库存状态 | 实时校验结果 | 库存不足时橙色提示，可触发"到货提醒" |
| VIN 校验标记 | ✅ 已通过 / ⚠️ 未校验 / ❌ 不匹配 | 不匹配时红色高亮，提供"查看适配件"链接 |
| 操作 | 删除 / 移入收藏 / 找替代件 | hover 显示操作按钮组 |

### 4.3 智能推荐引擎

| 推荐类型 | 触发条件 | 推荐逻辑 | 展示位置 |
|---------|---------|---------|---------|
| **补货组合推荐** | 购物车中有常规消耗品 | 基于经销商近 90 天下单频次，推荐遗漏的高频商品 | 购物车底部"你可能还需要"卡片组 |
| **运费凑单优化** | 当前金额未达免邮门槛 | 计算差额，推荐差价最小的热销品组合 | 购物车顶部横幅："再买 ¥XX 免运费" + 推荐商品横滑 |
| **替代件推荐** | 某商品缺货或库存不足 | 从 OMS 替代件关系表中匹配可用替代品 | 缺货商品行内嵌"推荐替代件"卡片 |
| **车型补全推荐** | VIN 码匹配的车型存在未购品类 | 基于该车型保养套餐推荐缺失品类 | 侧边栏"该车型推荐配件"面板 |

### 4.4 VIN 码批量校验

| 功能点 | 说明 |
|--------|------|
| **入口** | 购物车顶部工具栏"VIN 码校验"按钮 |
| **批量输入** | 支持粘贴多行 VIN 码（每行一个），或上传 Excel/CSV |
| **校验流程** | 逐条调用 OMS Vehicle Match API → 返回车型 + 适配结果 |
| **结果展示** | 表格形式：VIN码 / 解析车型 / 购物车中匹配商品数 / 不匹配商品数 |
| **快捷操作** | 不匹配商品一键"查看正确适配件" → 弹出替代件选择弹窗 |

### 4.5 结算摘要栏（右侧/底部固定）

| 字段 | 说明 |
|------|------|
| 已选商品数 | N 种商品，共 M 件 |
| 商品总额 | 原价合计 |
| 优惠减免 | 促销折扣 + 等级折扣明细 |
| 运费 | 实时计算；未达免邮门槛时显示差额 |
| 应付总额 | 加粗橙色大字 |
| 去结算按钮 | 橙色实心大按钮；未勾选商品时置灰 |

## 5. 结算下单模块

### 5.1 结算页布局

| 区域 | 内容 | 交互说明 |
|------|------|---------|
| **收货地址** | 默认地址卡片 + 切换/新增按钮 | 从个人中心地址簿拉取；支持按门店筛选 |
| **配送方式** | 快递配送 / 自提 / 指定物流公司 | 根据仓库位置和经销商偏好自动推荐 |
| **预计时效** | 预计发货时间 + 预计到货时间 | 调用 OMS 物流时效 API 实时计算 |
| **发票信息** | 发票抬头 + 税号 + 开票类型 | 从个人中心发票信息自动回填；支持修改 |
| **订单备注** | 文本域，最多 200 字 | 预填常用备注模板（如"请附合格证"） |
| **商品清单确认** | 精简版购物车列表（只读） | 可返回购物车修改 |
| **费用明细** | 商品金额 / 运费 / 优惠 / 应付总额 | 逐项展开明细 |
| **提交订单按钮** | 橙色实心大按钮 | 防重复点击；提交后跳订单详情页 |

### 5.2 地址管理规范

| 字段 | 必填 | 说明 |
|------|------|------|
| 收件人姓名 | ✅ | 支持从通讯录导入 |
| 联系电话 | ✅ | 手机号格式校验 |
| 省/市/区 | ✅ | 三级联动下拉 |
| 详细地址 | ✅ | 支持地图选点辅助填写 |
| 门店编号 | 选填 | 关联经销商门店体系 |
| 默认标记 | 选填 | 每个经销商可设 1 个默认地址 |
| 地址标签 | 选填 | 如"总仓""分店A""维修车间" |

### 5.3 订单确认与防错机制

| 校验项 | 触发时机 | 处理方式 |
|--------|---------|---------|
| 库存二次校验 | 点击"提交订单" | 调用 OMS 实时库存接口；不足时弹窗提示并允许调整 |
| 价格时效校验 | 点击"提交订单" | 若结算期间价格变动，弹窗告知新旧价差，确认后继续 |
| VIN 适配复核 | 提交时存在未校验 VIN 商品 | 黄色警告条提醒，可选择继续或返回校验 |
| 最小起订量 | 数量变更时 | 低于 MOQ 时禁止提交，红色提示 |
| 重复订单检测 | 提交后 | 30 分钟内相同 SKU+数量组合，弹窗"是否重复下单？" |

## 附录 A：状态机定义

### 订单状态流转

```
CREATED → WAIT_PAY → PAID → CONFIRMED → PICKING → SHIPPED → IN_TRANSIT → DELIVERED → SIGNED
   ↓          ↓                                        ↓
CANCELLED  PAY_TIMEOUT                             EXCEPTION → (人工介入) → RESOLVED
```

| 状态 | 中文名 | 经销商可操作 |
|------|--------|------------|
| CREATED | 已创建 | 去付款 / 取消 |
| WAIT_PAY | 待付款 | 去付款 / 取消 |
| PAID | 已付款 | 催发货 / 修改地址 / 取消 |
| CONFIRMED | 已确认 | 催发货 / 修改地址 |
| PICKING | 拣货中 | 查看进度 |
| SHIPPED | 已发货 | 查看物流 |
| IN_TRANSIT | 运输中 | 查看物流 / 确认收货 |
| DELIVERED | 已派送 | 确认收货 / 申请售后 |
| SIGNED | 已签收 | 再次购买 / 申请售后 |
| CANCELLED | 已取消 | 重新下单 |
| EXCEPTION | 异常 | 联系客服 / 查看处理进度 |

### 售后状态流转

```
SUBMITTED → REVIEWING → APPROVED → RETURN_WAITING → RETURNING → INSPECTING → REFUNDING → COMPLETED
    ↓           ↓                                                ↓
 CANCELLED   REJECTED ←———————————————————————(申诉)←—————— REJECTED
```

## 附录 B：跨系统交互边界

| 交互场景 | 云销通职责 | OMS 职责 | 接口方向 |
|---------|-----------|---------|---------|
| 商品浏览 | 渲染商品卡片/详情 | 提供商品主数据、价格、库存 | OMS → 云销通 |
| VIN 码匹配 | 采集 VIN 码、展示匹配结果 | 解析 VIN → 返回车型 → 过滤适配商品 | OMS → 云销通 |
| 加购/购物车 | 本地暂存 + UI 交互 | 校验库存、计算价格、返回推荐 | OMS → 云销通 |
| 下单 | 采集订单信息、提交请求 | 创建订单、锁定库存、触发履约 | 云销通 → OMS |
| 支付 | 对接支付渠道、回调通知 | 确认支付、推进订单状态 | 双向 |
| 物流追踪 | 展示轨迹时间轴 | 聚合物流商数据、计算预计时效 | OMS → 云销通 |
| 售后申请 | 采集售后信息、展示进度 | 审核、调度退换、退款执行 | 云销通 → OMS |
| 对账 | 展示对账单、提交差异反馈 | 生成对账单、处理差异、核销回款 | OMS → 云销通 |
| 消息推送 | 渲染消息列表、管理推送偏好 | 产生事件、推送消息 | OMS → 云销通 |
| 用户信息 | 展示/编辑个人资料 | 存储用户主数据、同步经销商等级 | 双向 |

> **铁律**：云销通不做任何业务判断（价格计算、库存扣减、状态流转、审核决策），所有写操作均为 API 透传，读操作以 OMS 实时返回为准。
