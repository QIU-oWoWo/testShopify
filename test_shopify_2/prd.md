# 项目需求：支付后地址二次确认 shopify App 的开发与上架

## 1. 项目目标
创建一个用于测试 Shopify App Store 上架流程的极简应用。该应用在支付成功页面嵌入一个地址编辑窗口，让用户可以查看并修改收货地址。所有交互均在前端完成，无需调用任何后端 API，**无需进行任何字段校验**。

## 2. 核心功能需求

### 2.1 功能位置
- 应用必须嵌入在 Shopify 的**支付成功页面（Order Status Page / Thank You page）**。
- 组件应显示在订单信息摘要的下方，作为一个独立的卡片区域。

### 2.2 数据来源
- 从当前订单上下文中自动获取**收货地址（shipping address）**数据。
- 需要获取的字段：收货人姓名、详细地址（行1）、城市、省份/州、国家/地区、邮政编码、电话号码。

### 2.3 界面状态与交互

**状态一：查看模式（默认）**
- 以清晰的只读文本形式展示完整的收货地址。
- 底部显示一个“编辑地址”按钮。

**状态二：编辑模式**
- 用户点击“编辑地址”后，所有地址字段切换为可编辑的输入框。
- 输入框内预先填入当前地址数据。
- 底部显示“确认地址”和“取消”两个按钮。
- 点击“取消”按钮：放弃所有更改，返回查看模式，地址恢复为原始数据。
- 点击“确认地址”按钮：直接接受当前输入框中的内容，**不进行任何校验**（包括空字段校验），更新卡片内显示的地址为新数据，退出编辑模式返回查看模式。

### 2.4 成功反馈
- 用户点击“确认地址”后，在卡片顶部显示一条简单的成功提示（例如“地址已更新”），提示在几秒后自动消失。
- **无需**对任何字段做必填、格式或长度校验，用户可保存任意内容（包括空值）。

### 2.5 数据存储说明
- **重要约束**：本次地址修改**仅在前端内存中进行**，不需要通过任何 API 将更改同步回 Shopify 后台或数据库。页面刷新后地址将恢复为原始订单数据。

## 3. 用户体验要求
- 界面风格简洁、清晰，符合 Shopify 后台的设计语言。
- 查看模式和编辑模式切换流畅，按钮响应明确。
- 成功提示信息友好即可。

## 4. 上架准备清单（后续需完成）
- 应用图标：1200x1200 像素图片。
- 应用名称与描述：合作伙伴后台填写，名称包含“地址确认”等关键词。
- 截图：准备 3-6 张功能截图，覆盖查看、编辑、确认成功等状态。
- 演示商店：准备包含测试订单的开发商店链接。
- 定价：设置为“免费”。

## 5. 交付标准
- 代码可正常运行于 Shopify 开发商店。
- 所有交互流程符合上述需求，无报错或异常中断。
- 代码包含清晰注释，便于后续理解和维护。
